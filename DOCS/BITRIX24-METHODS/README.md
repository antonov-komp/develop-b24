# Методологии работы с данными в Bitrix24

**Дата создания:** 2025-12-20 19:09 (UTC+3, Брест)  
**Версия:** 1.0  
**Описание:** Централизованная документация по всем методологиям работы с данными в Bitrix24

---

## Обзор

В Bitrix24 существует несколько методологий для работы с данными, каждая из которых предназначена для определённых сценариев использования. Данная документация структурирована по методологиям и категориям методов.

**Основной источник документации:** https://apidocs.bitrix24.ru/

---

## Методологии работы с данными

### 1. REST API
**Назначение:** Основной метод работы с данными в облачной версии Bitrix24  
**Документация:** https://context7.com/bitrix24/rest/  
**Папка:** [REST-API/](./REST-API/)

**Когда использовать:**
- Облачная версия Bitrix24
- Внешние интеграции
- Мобильные приложения
- Веб-приложения

**Особенности:**
- Работа через HTTP-запросы
- Использование вебхуков для авторизации
- Поддержка батч-запросов
- Ограничения по количеству запросов в секунду

**Основные категории методов:**
- CRM (лиды, сделки, контакты, компании)
- Задачи (tasks)
- Пользователи (users)
- Файлы (disk)
- Календарь (calendar)
- Мессенджер (im)
- Учёт времени (timeman)
- Placements (встройки в интерфейс)
- Webhooks (вебхуки)

---

### 2. D7 ORM (Object-Relational Mapping)
**Назначение:** Работа с данными в коробочной версии Bitrix24  
**Документация:** https://dev.1c-bitrix.ru/api_d7/  
**Папка:** [D7-ORM/](./D7-ORM/)

**Когда использовать:**
- Коробочная версия Bitrix24
- Внутренние модули Bitrix24
- Прямая работа с БД через ORM
- Высокая производительность

**Особенности:**
- Работа напрямую с базой данных
- Использование Entity-классов
- Поддержка связей (relations)
- Кеширование запросов

**Основные компоненты:**
- Entity-классы (DataManager)
- Query Builder
- Relations (связи между сущностями)
- Events (события)

---

### 3. SDK (Software Development Kit)
**Назначение:** Набор инструментов для разработки приложений под Bitrix24  
**Документация:** https://apidocs.bitrix24.ru/sdk/index.html  
**Папка:** [SDK/](./SDK/)

**Когда использовать:**
- Разработка приложений для Bitrix24
- Создание встроек (placements) в интерфейс
- Использование готовых UI-компонентов
- Работа с JavaScript API Bitrix24

**Особенности:**
- JavaScript SDK для работы с API
- UI Kit с готовыми компонентами
- Интеграция с интерфейсом Bitrix24
- Поддержка placements и вебхуков

**Основные компоненты:**
- JavaScript API (BX.*)
- UI Kit (кнопки, формы, таблицы, модальные окна)
- Placements API
- Webhooks API

---

### 4. MCP (Mobile Client Protocol)
**Назначение:** Протокол для мобильных приложений Bitrix24  
**Документация:** https://apidocs.bitrix24.ru/sdk/mcp.html  
**Папка:** [MCP/](./MCP/)

**Когда использовать:**
- Разработка мобильных приложений
- Синхронизация данных с мобильными устройствами
- Push-уведомления

**Особенности:**
- Оптимизирован для мобильных устройств
- Поддержка офлайн-режима
- Сжатие данных
- Инкрементальная синхронизация

## Выбор методологии

### Облачная версия Bitrix24
**Используйте:** REST API

**Причины:**
- Единственный способ работы с данными извне
- Поддержка вебхуков для авторизации
- Широкий набор методов

**Ограничения:**
- Лимиты на количество запросов
- Зависимость от доступности API
- Необходимость обработки ошибок сети

---

### Коробочная версия Bitrix24
**Используйте:** D7 ORM (для внутренних модулей) или REST API (для внешних интеграций)

**Причины:**
- D7 ORM — прямая работа с БД (высокая производительность)
- REST API — унификация с облачной версией

**Рекомендации:**
- Внутренние модули → D7 ORM
- Внешние интеграции → REST API
- Мобильные приложения → REST API или MCP

---

### Мобильные приложения
**Используйте:** MCP или REST API

**Причины:**
- MCP оптимизирован для мобильных устройств
- REST API — универсальный подход

**Рекомендации:**
- Нативные приложения → MCP
- Веб-приложения → REST API
