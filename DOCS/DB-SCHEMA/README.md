# Схема базы данных

**Дата создания:** 2025-12-19 11:52 (UTC+3, Брест)  
**Версия:** 1.0  
**Описание:** Описание структуры базы данных для REST приложения Bitrix24

---

## Общая концепция

REST приложение Bitrix24 работает с данными через **Bitrix24 REST API**. Данные хранятся в Bitrix24, приложение получает и отправляет их через API.

---

## Локальная база данных (если требуется)

Если приложение использует локальную БД для кеширования или хранения дополнительных данных, структура должна быть описана в `tables.md`.

### Структура документации

- `tables.md` — описание таблиц, столбцов, связей
- `indexes.md` — индексы и оптимизация

---

## Формат описания таблицы

```markdown
## Таблица `table_name`

**Назначение:** Описание назначения таблицы.  
**Пример использования:** Где и как используется таблица.

### Столбцы

1. **`id`**
   - Тип: `INT(11) AUTO_INCREMENT`
   - Ограничения: `PRIMARY KEY`
   - Описание: Уникальный идентификатор.
   - Пример: `12345`

2. **`name`**
   - Тип: `VARCHAR(255)`
   - Ограничения: `NOT NULL`
   - Описание: Название.
   - Пример: `Пример названия`

### Связи
- `table_name.foreign_key` → `other_table.id` (FOREIGN KEY)
- Индекс: `idx_table_name_column` (`column_name`)
```
