## Статусы задач

- **Новая** — задача создана, ожидает начала работы
- **В работе** — задача выполняется
- **На проверке** — задача выполнена, ожидает проверки
- **Завершена** — задача проверена и закрыта → **перемещается в ARCHIVE/**

---

## Архивация задач

### Когда архивировать
Задача архивируется после того, как:
1. Статус задачи изменён на **"Завершена"**
2. Все критерии приёмки выполнены
3. Задача проверена и принята

### Процесс архивации

1. **Обновить статус в задаче:**
   ```markdown
   **Статус:** Завершена
   **Дата завершения:** YYYY-MM-DD HH:MM (UTC+3, Брест)
   ```

2. **Добавить запись в историю правок:**
   ```markdown
   ## История правок
   - YYYY-MM-DD HH:MM (UTC+3, Брест): Задача завершена и заархивирована
   ```

3. **Переместить файл в ARCHIVE/:**
   ```bash
   mv TASKS/TASK-XXX-description.md TASKS/ARCHIVE/TASK-XXX-description.md
   ```

### Структура архива
- Все завершённые задачи хранятся в `TASKS/ARCHIVE/`
- Имя файла сохраняется без изменений
- Задачи в архиве остаются доступными для справки

## Приоритеты задач

- **Низкий** — задача не критична, может быть выполнена позже
- **Средний** — стандартная задача
- **Высокий** — задача важна, требует приоритетного выполнения
- **Критический** — задача блокирует работу, требует немедленного выполнения
