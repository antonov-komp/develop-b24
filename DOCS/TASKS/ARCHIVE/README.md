# Архив выполненных задач

**Дата создания:** 2025-12-19 11:52 (UTC+3, Брест)  
**Версия:** 1.0  
**Описание:** Архив завершённых задач для REST приложения Bitrix24

---

## Назначение

Папка `ARCHIVE/` содержит все задачи, которые были успешно выполнены и закрыты. Задачи перемещаются сюда из основной папки `TASKS/` после завершения.

---

## Процесс архивации

### Условия для архивации

Задача готова к архивации, когда:
1. ✅ Статус задачи: **"Завершена"**
2. ✅ Все критерии приёмки выполнены
3. ✅ Задача проверена и принята
4. ✅ В задаче указана дата завершения

### Шаги архивации

1. **Обновить статус в задаче:**
   ```markdown
   **Статус:** Завершена
   **Дата завершения:** YYYY-MM-DD HH:MM (UTC+3, Брест)
   ```

2. **Добавить финальную запись в историю:**
   ```markdown
   ## История правок
   - YYYY-MM-DD HH:MM (UTC+3, Брест): Задача завершена и заархивирована
   - [предыдущие записи...]
   ```

3. **Переместить файл:**
   ```bash
   # Из TASKS/ в TASKS/ARCHIVE/
   mv TASKS/TASK-XXX-description.md TASKS/ARCHIVE/TASK-XXX-description.md
   ```

## Структура архива

### Организация файлов

Все завершённые задачи хранятся в корне папки `ARCHIVE/`:
```
ARCHIVE/
├── README.md                    # Этот файл
├── TASK-001-lead-form.md       # Пример завершённой задачи
├── TASK-002-api-integration.md # Пример завершённой задачи
└── ...
```

### Именование файлов

Имя файла сохраняется без изменений:
- Формат: `TASK-XXX-description.md`
- Пример: `TASK-001-lead-form.md`

---

## Использование архива

### Для справки
Архив используется для:
- Поиска решений похожих задач
- Изучения реализованных подходов
- Анализа истории проекта
- Документирования выполненных работ

### Поиск в архиве
```bash
# Поиск задачи по номеру
find ARCHIVE/ -name "TASK-001-*"

# Поиск по описанию
grep -r "описание" ARCHIVE/

# Поиск по исполнителю
grep -r "Bitrix24 Программист" ARCHIVE/
```

## Статистика

### Подсчёт завершённых задач
```bash
# Количество задач в архиве
ls -1 ARCHIVE/*.md | wc -l

# Исключая README.md
ls -1 ARCHIVE/TASK-*.md | wc -l
```

## Правила работы с архивом

### ✅ Можно
- Просматривать задачи для справки
- Использовать код из задач как примеры
- Копировать структуру задач для новых

### ❌ Нельзя
- Удалять задачи из архива (без согласования)
- Изменять содержимое задач (только для исправления ошибок)
- Перемещать задачи обратно в TASKS/ (без веской причины)
